# 双系统整合说明文档

## 🔄 系统整合概述

本次更新将武术系统和修仙系统进行了深度整合，实现了统一的修炼值获取机制和状态管理体系。

## 🎯 主要整合内容

### 1. 统一修炼定时任务
- **修仙系统**：通过打坐和盔甲架冥想获得修为和灵力
- **武者系统**：通过战斗训练和实战获得修为和内力
- **共同特征**：两者都支持自然状态下的缓慢资源恢复

### 2. 双系统状态管理
- 使用 `/status` 命令查看完整的双系统状态
- 同时显示修仙境界、武者境界、资源数值和活动状态
- 支持多语言显示（中文/英文）

### 3. 资源恢复机制
- **非活动状态**：灵力和内力都会缓慢自然恢复
- **修仙活动**：打坐时主要恢复灵力，略微恢复内力
- **武者活动**：战斗时消耗内力，非战斗训练时恢复内力

## ⚙️ 技术实现细节

### 修炼值获取逻辑
```java
// 修仙修为获取（打坐/冥想）
if (isMeditating || isRiding) {
    double gain = baseGain * multiplier;
    playerData.addExp(gain);           // 旧系统
    enhancedPlayerData.addXiuzhenExp(gain); // 新系统
}

// 武者修为获取（战斗训练）
if (isInCombatTraining || isInCombat) {
    double gain = baseGain * 0.7 * (isInCombat ? 1.5 : 1.0);
    enhancedPlayerData.addWushuExp(gain);
}
```

### 资源管理系统
- **灵力**：主要用于修仙功法，打坐时恢复
- **内力**：主要用于武者功法，战斗时消耗，休息时恢复
- **自然恢复**：非活动状态下两种资源都会缓慢恢复

## 🎮 使用指南

### 查看状态
```
/status              # 查看双系统完整状态
/xiuzhen stats       # 查看修仙系统状态
/wushu stats         # 查看武者系统状态
```

### 修炼方式
**修仙修炼：**
```
/xiuzhen meditate    # 开始打坐修炼
/ride armor_stand    # 骑乘盔甲架冥想（需要特定标签）
```

**武者修炼：**
```
/wushu train         # 开始战斗训练
/engage combat       # 参与战斗获得修为
```

## 🔧 配置说明

### 修炼增益设置
```yaml
settings:
  cultivation:
    base_gain_per_second: 0.05    # 基础修为增长
    move_distance_threshold: 0.5  # 移动中断阈值
    
    armor_stand_meditation:
      enabled: true               # 盔甲架冥想开关
      multiplier: 2.0             # 冥想增益倍数
```

### 战斗训练设置
```yaml
wushu_system:
  combat_gain_multiplier: 1.5     # 战斗时修为增益
  training_gain_multiplier: 1.0   # 训练时修为增益
  neili_consumption_rate: 0.5     # 战斗内力消耗率
  neili_recovery_rate: 0.3        # 非战斗内力恢复率
```

## 📊 系统平衡性

### 修为获取平衡
- **修仙系统**：稳定但较慢的增长速度
- **武者系统**：战斗时快速增长，平时缓慢增长
- **总体平衡**：鼓励玩家根据喜好选择专精方向

### 资源管理平衡
- 灵力和内力有各自的用途和恢复机制
- 双修玩家可以获得更全面的能力
- 专精玩家在特定领域更有优势

## 🐛 已知问题与解决方案

### 问题1：战斗检测不够准确
**解决方案**：通过最后战斗时间戳来判断，5秒内视为战斗状态

### 问题2：资源恢复可能过快
**解决方案**：设置了合理的恢复速率和消耗比例

### 问题3：状态显示信息过多
**解决方案**：使用格式化消息和颜色编码提高可读性
