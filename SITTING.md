# 修仙插件坐下兼容功能说明

## 🪑 功能概述

现在玩家可以通过第三方插件的坐下命令来自动进入打坐修炼状态，享受更高的修为获取效率。

## 🎯 支持的插件

### 已兼容插件
- **CMI** - 使用 `/cmi sit` 和 `/cmi stand`
- **EssentialsX** - 使用 `/e sit` 和 `/e stand`  
- **Sit插件** - 使用 `/sit` 和 `/stand`
- **原生Minecraft** - 在楼梯、台阶等方块上自动识别

## ⚙️ 配置选项

### config.yml 设置
```yaml
settings:
  cultivation:
    seat_meditation:
      # 是否启用坐下自动打坐
      enabled: true
      
      # 坐下时的修为增益倍数
      multiplier: 1.5
      
      # 坐下检测频率（ticks）
      detection_interval: 10
      
      # 起立冷却时间（ticks）
      stand_cooldown: 40
```

## 🎮 使用方法

### 玩家操作
1. **坐下修炼**：
   - 使用 `/sit` 命令坐下
   - 或直接坐在楼梯、台阶上
   - 系统自动识别并开始打坐

2. **起立停止**：
   - 使用 `/stand` 命令起立
   - 或直接站起来
   - 系统自动停止打坐状态

### 修为增益
- **普通打坐**：基础修为获取速度
- **坐下修炼**：修为获取速度 × 1.5（可配置）
- **同时享受**：粒子效果 + Action Bar提示

## 📊 功能特性

### 智能检测
- **多方块支持**：楼梯、台阶、地毯、睡莲等
- **状态同步**：与原有打坐系统完美融合
- **防抖动**：起立冷却机制避免频繁切换

### 用户体验
- **自动提示**：坐下时显示"坐着修炼中..."提示
- **视觉反馈**：独特的粒子效果区分不同修炼方式
- **无缝切换**：可在打坐和坐下间自由切换

### 系统集成
- **事件监听**：实时监控玩家坐下/起立状态
- **数据同步**：修炼进度在不同状态下保持连续
- **性能优化**：高效的检测算法，不影响服务器性能

## 🔧 技术实现

### 核心组件
- **SeatListener.java** - 座位状态监听器
- **ThirdPartyIntegration.java** - 第三方插件集成器
- **增强的修炼任务** - 支持多种修炼方式

### 检测机制
1. **方块检测**：识别可坐下的方块类型
2. **状态检测**：监控玩家潜行和移动状态
3. **插件API**：与第三方插件进行命令交互

## ⚠️ 注意事项

1. **兼容性**：确保相关插件已正确安装并启用
2. **权限设置**：玩家需要有使用坐下命令的权限
3. **配置调整**：可根据服务器需求调整增益倍数
4. **性能监控**：大量玩家同时坐下时注意服务器负载

## 🎨 自定义选项

### 增益倍数调整
```yaml
multiplier: 1.2  # 20%增益
multiplier: 2.0  # 100%增益（双倍）
multiplier: 1.0  # 无增益（与普通打坐相同）
```

### 检测频率优化
```yaml
detection_interval: 5   # 更敏感（每0.25秒检测）
detection_interval: 20  # 更节省性能（每1秒检测）
```

这个功能让修仙体验更加自然和多样化，玩家可以选择自己喜欢的修炼姿势！
