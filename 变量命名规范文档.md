# SimpleXiuzhen 插件变量命名规范文档

## 概述
本文档整理了插件中所有核心变量的命名、用途和数据示例，用于统一开发规范，避免命名混乱。

---

## 一、PlayerData 类变量（玩家数据核心类）

### 1.1 基础标识变量
| 变量名 | 类型 | 用途 | 示例值 |
|--------|------|------|--------|
| `playerUUID` | UUID | 玩家的唯一标识符 | `e28d96a9-ef92-486c-9eff-eaa0f25060cd` |
| `plugin` | SimpleXiuzhen | 插件主类实例引用 | `SimpleXiuzhen@1234` |
| `configManager` | ConfigManager | 配置管理器引用 | `ConfigManager@5678` |

### 1.2 修仙系统变量
| 变量名 | 类型 | 用途 | 示例值 | 配置路径 |
|--------|------|------|--------|----------|
| `currentRealmKey` | String | 当前境界的键值（英文） | `"LianQi"`, `"ZhuJi"`, `"JinDan"` | `xiuzhen.current_realm` |
| `currentSubLevelIndex` | int | 当前段位索引（从 0 开始） | `0`(一层), `1`(二层), `8`(九层) | `xiuzhen.current_sub_level_index` |
| `totalExp` | double | **累积总修为**（用于判断境界） | `10999.0`, `5000.0` | `xiuzhen.total_exp` |
| `hasBrokenThrough` | boolean | 当前境界是否已突破（未突破不能获得修为） | `true`, `false` | `xiuzhen.has_broken_through` |
| `currentLingli` | double | 当前灵力值 | `119.0`, `200.0` | `xiuzhen.current_lingli` |
| `maxLingli` | double | 最大灵力值（随境界提升） | `200.0`, `500.0` | `xiuzhen.max_lingli` |

### 1.3 武者系统变量
| 变量名 | 类型 | 用途 | 示例值 | 配置路径 |
|--------|------|------|--------|----------|
| `currentWushuRealmKey` | String | 当前武者境界的键值 | `"XiuLian"`, `"TongMai"`, `"ZhuJing"` | `wushu.current_realm` |
| `currentWushuSubLevelIndex` | int | 当前武者段位索引 | `0`(初学), `1`(一层), `4`(五层) | `wushu.current_sub_level_index` |
| `totalWushuExp` | double | **武者累积总修为** | `10949.0`, `500.0` | `wushu.total_exp` |
| `hasWushuBrokenThrough` | boolean | 武者境界是否已突破 | `true`, `false` | `wushu.has_broken_through` |
| `currentNeili` | double | 当前内力值 | `104.0`, `200.0` | `wushu.current_neili` |
| `maxNeili` | double | 最大内力值 | `200.0`, `500.0` | `wushu.max_neili` |

### 1.4 状态变量
| 变量名 | 类型 | 用途 | 示例值 | 配置路径 |
|--------|------|------|--------|----------|
| `isMeditating` | boolean | 是否在打坐（修仙） | `true`, `false` | `status.is_meditating` |
| `isInCombatTraining` | boolean | 是否在武者训练 | `true`, `false` | `status.is_combat_training` |
| `lastLocation` | Location | 上次记录的位置（检测移动） | `Location{world=world,x=100.5,y=64.0,z=200.5}` | 不保存 |
| `lastCombatTime` | long | 上次战斗时间戳（毫秒） | `1709625600000` | `status.last_combat_time` |

## 二、配置文件变量（realms.yml）

### 2.1 修仙境界配置结构
```yaml
realms:
  LianQi:  # 境界 Key（英文，用于代码）
    display_name: "炼气"  # 显示名称（中文，用于 UI）
    level: 1  # 境界层级
    permission: "simplexiuzhen.realm.lianqi"  # 权限节点
    breakthrough_requirements:  # 突破到下一境界的材料需求
      items:
        - material: IRON_INGOT  # 材料类型
          amount: 10  # 数量
        - material: GOLD_INGOT
          amount: 5
    sub_levels:  # 小境界列表
      - name: "一层"  # 段位名称
        required_exp: 0  # 到达此段位需要的**总修为**
      - name: "二层"
        required_exp: 50
      - name: "三层"
        required_exp: 150
```

### 2.2 配置变量对照表
| 配置键 | 含义 | 示例值 | 用途 |
|--------|------|--------|------|
| `realms.LianQi.display_name` | 炼气境界显示名称 | `"炼气"` | UI 显示 |
| `realms.LianQi.level` | 境界层级 | `1` | 计算最大灵力 |
| `realms.LianQi.sub_levels[0].name` | 第一段位名称 | `"一层"` | UI 显示 |
| `realms.LianQi.sub_levels[0].required_exp` | 到达一段位需要的总修为 | `0` | 境界判断 |
| `realms.LianQi.sub_levels[1].required_exp` | 到达二段位需要的总修为 | `50` | 境界判断 |
| `realms.LianQi.breakthrough_requirements.items` | 突破材料列表 | `[IRON_INGOT:10, GOLD_INGOT:5]` | 突破检查 |

## 五、重要方法命名规范

### 5.1 Getter 方法
| 方法名 | 返回类型 | 用途 |
|--------|---------|------|
| `getCurrentRealmKey()` | String | 获取当前修仙境界 Key |
| `getCurrentSubLevelIndex()` | int | 获取当前修仙段位索引 |
| `getTotalExp()` | double | 获取修仙总修为 |
| `hasBrokenThrough()` | boolean | 检查是否已突破 |
| `getCurrentLingli()` | double | 获取当前灵力 |
| `getMaxLingli()` | double | 获取最大灵力 |
| `getCurrentWushuRealmKey()` | String | 获取当前武者境界 Key |
| `getTotalWushuExp()` | double | 获取武者总修为 |
| `hasWushuBrokenThrough()` | boolean | 检查武者是否已突破 |

### 5.2 Setter 方法
| 方法名 | 参数 | 用途 |
|--------|------|------|
| `setCurrentRealmKey(String)` | realmKey | 设置修仙境界 |
| `setCurrentSubLevelIndex(int)` | index | 设置修仙段位索引 |
| `setTotalExp(double)` | exp | 设置修仙总修为 |
| `setBrokenThrough(boolean)` | brokenThrough | 设置突破状态 |
| `performXiuzhenBreakthrough(String)` | nextRealm | 执行修仙大境界突破 |
| `confirmXiuzhenBreakthrough()` | - | 确认突破完成（恢复修为获取） |

### 5.3 修为操作方法
| 方法名 | 参数 | 用途 |
|--------|------|------|
| `addExp(double)` | amount | 增加修仙修为 |
| `addWushuExp(double)` | amount | 增加武者修为 |
| `consumeLingli(double)` | amount | 消耗灵力 |
| `consumeNeili(double)` | amount | 消耗内力 |
| `recoverLingli(double)` | amount | 恢复灵力 |
| `recoverNeili(double)` | amount | 恢复内力 |

### 5.4 境界检查方法
| 方法名 | 参数 | 用途 |
|--------|------|------|
| `checkXiuzhenLevelUp(Player)` | player | 检查修仙小境界突破 |
| `checkWushuLevelUp(Player)` | player | 检查武者小境界突破 |
| `recalculateRealmFromExp()` | - | 根据总修为重新计算境界 |
| `attemptXiuzhenRealmBreakthrough()` | - | 尝试修仙大境界突破（检查材料） |

## 八、命名规范总结

### 8.1 前缀规范
- **修仙系统**：使用 `xiuzhen` 或 `xz` 前缀
- **武者系统**：使用 `wushu` 或 `ws` 前缀
- **灵力**：使用 `lingli` 或 `lingli`
- **内力**：使用 `neili` 或 `neili`

### 8.2 总修为命名
- ✅ `totalExp` - 累积总修为（用于境界判断）
- ❌ `currentExp` - 已废弃（容易误解为当前段位修为）

### 8.3 境界相关命名
- ✅ `RealmKey` - 境界的英文键值（如 `LianQi`）
- ✅ `RealmName` - 境界的中文显示名（如 `炼气`）
- ✅ `SubLevelIndex` - 段位索引（从 0 开始）
- ✅ `SubLevelName` - 段位显示名（如 `一层`）

### 8.4 突破相关命名
- ✅ `hasBrokenThrough` - 是否已突破（布尔值）
- ✅ `performBreakthrough` - 执行突破（动词）
- ✅ `confirmBreakthrough` - 确认突破完成

## 九、常见错误示例

### 9.1 错误用法
```java
// ❌ 错误：使用 currentExp 容易误解
double exp = playerData.getCurrentExp();

// ❌ 错误：混淆修仙和武者
playerData.addExp(100);  // 不明确是修仙还是武者

// ❌ 错误：直接使用数字索引而不说明
int index = 2;  // 这是什么段位？
```

### 9.2 正确用法
```java
// ✅ 正确：使用 totalExp 明确表示总修为
double totalExp = playerData.getTotalExp();

// ✅ 正确：明确区分系统
playerData.addExp(100);        // 修仙修为
playerData.addWushuExp(100);   // 武者修为

// ✅ 正确：使用有意义的变量名
int currentSubLevelIndex = playerData.getCurrentSubLevelIndex();
```

## 十、配置文件更新清单

### 10.1 需要更新的文件
1. `src/main/resources/realms.yml` - 修仙境界配置
2. `src/main/resources/martial_realms.yml` - 武者境界配置
3. `src/main/resources/config.yml` - 主配置文件
4. `src/main/resources/lang/zh_cn.yml` - 中文语言文件
5. `src/main/resources/lang/en_us.yml` - 英文语言文件

### 10.2 配置更新后的操作
1. 停止服务器
2. 删除 `plugins/SimpleXiuzhen/` 下的旧配置文件
3. 重启服务器生成新配置文件
4. 根据需要调整配置值
5. 再次重启服务器
